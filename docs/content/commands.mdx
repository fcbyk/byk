# 内置命令

byk 提供一组内置子命令，用于初始化功能、管理插件、生成补全脚本等。运行 `byk -h` 或不带任何参数即可看到帮助信息。

## init

初始化 byk 的各项功能。byk 不会自动创建任何配置，所有功能都需要用户手动按需初始化。

```bash
byk init <feature>
```

| Feature | 说明 |
|---------|------|
| `npm` | 初始化 npm 包管理环境，创建 `~/.byk/node-pkgs/` 目录及 `ni`/`nu` 别名 |
| `pnpm` | 初始化 pnpm 包管理环境，同上，但使用 pnpm 作为包管理器 |
| `comp` | 初始化 shell 补全（自动检测 zsh/bash），在对应 rc 文件中追加补全脚本 |
| `cache` | 初始化 CLI 家目录 `~/.byk/` 及缓存目录结构（alias、cache、logs、plugins） |

```bash title="示例"
byk init npm       # 初始化 npm 环境
byk init comp      # 配置 shell 补全
byk init cache     # 创建家目录和缓存
```

:::note
`init` 操作是幂等的——已存在的目录或已配置的补全会被跳过，不会重复创建。
:::

## add

从远程仓库或本地文件安装插件。

```bash
byk add [OPTIONS] <NAME>
```

| 选项 | 说明 |
|------|------|
| `--branch <NAME>` | 指定仓库分支（默认 `main`） |
| `--file <PATH>` | 使用本地 `byk.json` 文件代替远程仓库 |
| `-e, --editable <DIR>` | 可编辑安装（`pip install -e <DIR>`，读取 `<DIR>/byk.json`） |

插件来源支持三种模式：

- **中心仓库**：`byk add hello` — 从 `fcbyk/byk-plugins` 中心仓库安装
- **社区仓库**：`byk add user/repo` 或 `byk add user/repo/key` — 从指定 GitHub 仓库安装
- **本地安装**：`byk add --file ./local.json my-key` 或 `byk add -e .`

```bash title="示例"
byk add hello                     # 从中心仓库安装
byk add user/repo/key             # 从社区仓库安装指定 key
byk add user/repo                 # 安装社区仓库 byk.json 中的第一个 key
byk add --branch dev hello        # 从 dev 分支安装
byk add --file ./local.json key   # 使用本地 byk.json 安装
byk add -e .                      # 可编辑安装当前目录
```

:::tip
首次运行 `byk add` 时，如果 Python venv 尚未初始化，会提示是否自动创建。
:::

## remove

移除已初始化的功能或卸载插件。

```bash
byk remove <feature>
```

| Feature | 说明 |
|---------|------|
| `comp` | 移除 shell 补全配置（从 .zshrc / .bashrc 中删除相关行） |
| `node` | 移除 node-pkgs 环境、别名文件和缓存 |
| `all` | 移除所有持久化数据（`~/.byk/` 和 shell 补全），需输入确认 |
| `<plugin-key>` | 卸载指定插件（执行 `pip uninstall -y` 并清除命令缓存） |

```bash title="示例"
byk remove comp        # 移除 shell 补全
byk remove node        # 移除 node-pkgs 环境
byk remove hello       # 卸载名为 hello 的插件
byk remove all         # 清除所有数据
```

:::note
如果当前运行的 byk 二进制文件位于 `~/.byk/` 下（如通过 curl 脚本安装），`remove all` 会跳过自身所在目录，并提示手动删除。
:::

## completion

生成指定 shell 的补全脚本并输出到 stdout。通常由 `byk init comp` 自动调用，无需手动使用。

```bash
byk completion <shell>
```

支持的 shell：`zsh`、`bash`、`fish`。

```bash title="手动配置补全"
# zsh
source <(byk completion zsh)

# bash
source <(byk completion bash)

# fish
byk completion fish | source
```

## 全局选项

| 选项 | 说明 |
|------|------|
| `-v, --version` | 打印版本号、构建信息及安装路径 |
| `-h, --help` | 打印帮助信息（含 Commands、NPM Commands、Aliases 一览） |
| `--info [TOPIC]` | 查看系统信息面板 |

### --info 详解

`--info` 是一个多功能信息查询入口，支持三种模式：

```bash
byk --info              # 总览面板：目录路径、缓存状态、补全、Node、Python
byk --info doctor       # 诊断检查：缓存健康、配置校验、冲突检测
byk --info plugins      # 已安装插件列表及其命令详情
byk --info <name>       # 命令名全量路由查询（查找内置命令、插件、NPM、别名中的所有匹配）
```

```bash title="命令名查询示例"
byk --info build
# 输出所有名为 "build" 的命令来源：
# Built-in / Plugin / NPM / Alias（含优先级、模板、占位符等详情）
```

## 命令执行优先级

当输入 `byk <name>` 时，byk 按以下顺序匹配命令：

1. **内置子命令** — `init`、`add`、`remove`、`completion`
2. **全局选项** — `-v`、`-h`、`--info`
3. **插件命令** — 通过 `byk add` 安装的 Python 插件
4. **NPM 命令** — 通过 `byk init npm/pnpm` 安装的 npm 包
5. **精确执行语法** — `@file.key`（本地）/ `@@file.key`（全局）
6. **别名** — 从 `*.byk.json` 文件中加载的别名

匹配到第一个命中即执行，不会继续向下查找。

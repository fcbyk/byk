# 插件系统

byk 通过隔离的 Python 虚拟环境运行插件，不污染系统全局的 pip 包。插件基于 Python 模块执行，安装后即可作为一级命令使用。

## 工作原理

插件系统依赖 `~/.byk/venv/` 下的 Python 虚拟环境。当你运行 `byk add <user/repo[/key]>` 时：

1. 从远程仓库（或本地文件）拉取插件协议 `byk.json`
2. 查找匹配的插件 key，解析其行为（`py-m` 或 `py-f`）
3. 在隔离的 venv 中执行 `pip install`（`py-m`）或下载/拷贝脚本文件（`py-f`）
4. 将命令路由持久化到 `~/.byk/plugins/plugins.cmd.json`，包追踪信息持久化到 `~/.byk/plugins/plugins.pkg.json`

之后运行 `byk <command>` 时，byk 根据 `behavior` 字段区分执行方式：
- `py-m`：执行 `python -m <target>`
- `py-f`：执行 `python <scripts_dir>/<target>`

## 初始化

首次使用插件功能时，venv 不存在则 byk 会自动提示创建，也可以手动初始化：

```bash
byk add <any-plugin>   # 首次 add 时自动创建 venv
```

## 安装插件

### 从远程仓库安装

任何 GitHub 仓库只需在根目录放一个 `byk.json` 即可作为 byk 插件源：

```bash
byk add user/repo           # 安装该仓库 byk.json 中的第一个 key
byk add user/repo/my-key    # 安装指定 key
```

### 指定分支

```bash
byk add --branch dev hello   # 从 dev 分支的 byk.json 拉取
```

### 本地安装

```bash
byk add --file ./local.json my-key   # 使用本地 byk.json 文件
```

### 可编辑安装

适合插件开发场景，将本地源码以可编辑模式安装到 venv：

```bash
byk add -e .              # 从当前目录可编辑安装（读取 ./byk.json）
byk add -e /path/to/src   # 从指定目录可编辑安装
```

## 运行插件

安装完成后，插件命令自动注册为 byk 的一级命令，直接使用即可：

```bash
byk <command> [args...]
```

所有命令行参数会原样透传给底层的 Python 模块。

```bash title="示例"
byk lansend --target 192.168.1.10 "hello"
# 等价于 venv/bin/python -m byklansend.main --target 192.168.1.10 "hello"
```

## 查看已安装插件

```bash
byk --info plugins      # 列出所有已安装插件及其命令详情
byk --info <command>    # 查询某个命令是否来自插件（含模块路径、描述等）
```

## 卸载插件

```bash
byk remove <plugin-key>   # 卸载指定插件
```

卸载流程：在 venv 中执行 `pip uninstall -y <package>`，然后清除 `plugins.cmd.json` 和 `plugins.pkg.json` 中该插件注册的所有命令。

## 目录结构

插件相关的数据存储在以下位置：

```tree
~/.byk/
├── venv/               # Python 虚拟环境（python3 -m venv 创建）
│   ├── bin/            # Python 解释器和 pip
│   └── lib/
└── plugins/
    ├── plugins.cmd.json   # 命令路由（热路径：命令解析、帮助渲染）
    ├── plugins.pkg.json   # 包追踪（冷路径：卸载、信息查询）
    └── scripts/           # py-f 脚本文件存放目录
```

## 状态文件

插件系统使用两个 JSON 文件分别管理命令路由和包追踪，由 `byk add` / `byk remove` 自动维护。

### plugins.cmd.json（命令路由）

热路径文件，用于命令解析、路由和帮助渲染：

```json
{
  "commands": {
    "lansend": {
      "behavior": "py-m",
      "target": "byklansend.main",
      "description": "Send messages via LAN"
    },
    "myscript": {
      "behavior": "py-f",
      "target": "myscript.py",
      "description": "My custom script"
    }
  },
  "python_executable": "/Users/coke/.byk/venv/bin/python"
}
```

| 字段 | 说明 |
|------|------|
| `commands` | 命令名 → 行为类型、目标、描述的映射 |
| `python_executable` | venv 内 Python 解释器路径，用于执行插件 |
| `behavior` | 行为类型：`py-m`（模块）或 `py-f`（脚本） |
| `target` | 行为目标：`py-m` 为模块路径，`py-f` 为脚本文件名 |

### plugins.pkg.json（包追踪）

冷路径文件，记录每个插件的安装包和注册命令，用于卸载和信息查询：

```json
{
  "packages": {
    "lansend": {
      "source": "user/repo",
      "py_m": {
        "name": "byk-lansend",
        "commands": ["lansend"]
      }
    },
    "myscript": {
      "source": null,
      "py_f": {
        "scripts": ["myscript.py"],
        "commands": ["myscript"],
        "dependencies": ["requests>=2.28"]
      }
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `packages` | 插件 key → 包信息的映射 |
| `source` | 插件来源（`user/repo` 格式），本地安装时为 `null` |
| `py_m` | `py-m` 行为记录：pip 包名和注册的命令列表 |
| `py_f` | `py-f` 行为记录：脚本文件名、命令列表和依赖列表 |

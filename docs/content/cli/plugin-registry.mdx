# 插件协议 byk.json

每个 byk 插件的发布入口是仓库根目录下的 `byk.json` 文件。它定义了插件的安装方式、下载资源和注册的命令。本文档面向插件发布者，说明如何编写这个文件。

## 文件位置

将 `byk.json` 放在 GitHub 仓库的根目录。byk 通过 `raw.githubusercontent.com` 拉取该文件：

```
https://raw.githubusercontent.com/{owner}/{repo}/{branch}/byk.json
```

## 设计哲学

`byk.json` 是一个**过程式操作脚本**，按顺序声明操作块：

```
1. pip / pip-keep    → 安装 Python 依赖到虚拟环境
2. downloads.scripts  → 下载/拷贝脚本文件到 plugins/scripts/
3. downloads.bin      → 下载二进制文件到 plugins/bin/（$tar 标记区分直下/解压）
4. download-scripts   → 语法糖，等价于 downloads.scripts（扁平写法）
5. download-bin       → 语法糖，等价于 downloads.bin（扁平写法）
6. command            → 注册单个命令（命令名 = 插件 key）
7. commands           → 注册多个命令到路由表
```

> **推荐**：单命令插件用 `command`，多命令插件用 `commands`。两者可以共存，但不要在同名 key 上重复定义。

JSON key 的声明顺序即执行顺序，阅读即执行。

## 整体结构

`byk.json` 是一个 JSON 对象，每个顶级 key 为一个**插件 key**（即 `byk add <key>` 中的 `<key>`），值为该插件的操作定义：

```json
{
  "$default": "<plugin-key>",
  "<plugin-key>": {
    "pip": [...],
    "commands": { ... }
  }
}
```

一个 `byk.json` 可以包含多个插件 key，用户通过 `byk add <key>` 指定要安装哪个。

### $default

`$default` 字段指定当用户不指定 key 时（`byk add user/repo`）默认安装哪个插件。

```json
{
  "$default": "pym",
  "pym": "./plugins/py-module/byk.json",
  "pys": "./plugins/py-script/byk.json"
}
```

| 场景 | 行为 |
|------|------|
| 有 `$default` | 使用 `$default` 指定的 key |
| 无 `$default`，只有一个有效 key | 自动使用该 key（无需显式声明） |
| 无 `$default`，多个有效 key | 报错，提示可用 key 列表 |

:::note

- `$default` 的值必须是 `byk.json` 中存在的有效 key，否则安装时报错
- 只有一个 key 时无需声明 `$default`，byk 会自动推断
- 有效 key 指不以 `$` 开头的顶级字段

:::

### $var

`$var` 字段用于定义可复用的变量，减少路径或 URL 中的重复字符串。定义后可在任意位置的字符串值中使用 `{变量名}` 语法引用。

```json
{
  "$var": {
    "p": "./plugins"
  },
  "pym": "{p}/py-module/byk.json",
  "pys": "{p}/py-script/byk.json",
  "pyb": "{p}/py-bin/byk.json"
}
```

上例中三个 plugin key 的路径前缀 `./plugins` 被提取到 `$var.p`，通过 `{p}` 引用，**等价于**：

```json
{
  "pym": "./plugins/py-module/byk.json",
  "pys": "./plugins/py-script/byk.json",
  "pyb": "./plugins/py-bin/byk.json"
}
```

变量也常用于减少 `pip` 中的重复 URL：

```json
{
  "$var": {
    "url": "https://github.com/fcbyk/byk/releases/download/assets"
  },
  "pym": {
    "pip": [
      "byk_pym @ {url}/byk_pym-2026.6.21-py3-none-any.whl"
    ],
    "commands": { ... }
  }
}
```

#### 规则

| 规则 | 说明 |
|------|------|
| 作用域 | 仅限当前 `byk.json` 文件，**不穿透到 Ref 引用的文件** |
| 替换目标 | 对原始 JSON 字符串逐变量替换，覆盖任意位置的字符串值 |
| 未定义变量 | 静默保留原文，用户自行负责 |
| `$var` 值类型 | 必须是字符串，不接受数字、布尔、数组等 |

#### 处理顺序

```
byk.json 文件
  → 1. 解析 JSON，提取 $var
  → 2. 在原始 JSON 字符串上逐变量替换 {var} 占位符
  → 3. 解析 Ref 引用（已替换为完整路径）
  → 4. Ref 引用的文件回到步骤 1（独立的 $var 作用域）
```

:::note

- `$var` 的 value 同样参与替换，交叉引用（如 `"a": "{b}/foo"`）会逐变量传递替换
- `$var` 不会传递到 Ref 引用的子文件，每个 `byk.json` 拥有独立的变量作用域
- 变量替换发生在 Ref 解析之前，因此 `"{p}/py-module/byk.json"` 会先变成 `"./plugins/py-module/byk.json"`，再作为相对路径被解析

:::

## 操作块

### pip

通过 pip 安装 Python 包到 byk 的隔离虚拟环境，**卸载插件时自动执行 `pip uninstall`**。

```json
{
  "hello": {
    "pip": ["byk-hello"],
    "commands": { ... }
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pip` | string[] | 否 | pip 包名或 URL 列表，卸载时自动清理 |
| `pip-keep` | string[] | 否 | pip 包名或 URL 列表，卸载时保留（共享依赖） |

`pip` / `pip-keep` 数组中的每一项可以是：
- **包名**：`"rich"`、`"click>=8.0"`、`"requests==2.28.0"` — 从 PyPI 安装
- **URL**：`"https://github.com/.../xxx.whl"` — 从远程 URL 安装
- **name @ url**：`"my-pkg @ https://github.com/.../xxx.whl"` — 指定包名 + URL，卸载时用包名执行 `pip uninstall`

> **纯 URL 无法卸载**：如果 `pip` 中写的是纯 URL（无 `name @` 前缀），卸载时静默跳过，因为无法知道包名。如需自动卸载，请使用 `"name @ url"` 格式。

```bash
byk add user/repo/hello
```

#### 示例：专属包 + 共享依赖

```json
{
  "hello": {
    "pip": [
      "my-hello @ https://github.com/fcbyk/byk/releases/download/plugins/byk_hello-2026.6.21-py3-none-any.whl"
    ],
    "pip-keep": ["rich", "click>=8.0"],
    "commands": { ... }
  }
}
```

- `my-hello` 是插件专属包，卸载时自动 `pip uninstall my-hello`
- `rich`、`click` 是共享依赖，卸载时保留

### downloads

`downloads` 是下载区块，统一管理要下载到 `plugins/` 目录的文件。包含两个子区块：

- **`scripts`**：下载/拷贝脚本文件到 `plugins/scripts/`
- **`bin`**：下载二进制文件到 `plugins/bin/`，通过 `$tar` 标记区分直接下载和压缩包解压

#### downloads.scripts

将远程或本地的脚本文件下载/拷贝到 `plugins/scripts/` 目录。使用 `map` 对象，**key 为保存的文件名**（可自定义任意名称），value 为远程 URL 或相对路径。

```json
{
  "hello": {
    "downloads": {
      "scripts": {
        "runner.py": "./scripts/runner.py",
        "util.py": "https://example.com/util.py"
      }
    },
    "commands": { ... }
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `downloads.scripts` | object | 否 | 脚本下载映射：`{ "<文件名>": "<远程URL或相对路径>" }` |

`scripts` 中的每个键值对：
- **key**：保存到 `plugins/scripts/<key>` 的文件名（可含后缀或无后缀）
- **value**：远程 URL（自动下载）或相对路径 `./xxx`（自动拷贝），解析规则同 Ref 引用

#### 与 commands 的关系

```json
{
  "jiahao": {
    "downloads": {
      "scripts": {
        "jiahao.py": "https://raw.githubusercontent.com/fcbyk/pyodd/refs/heads/main/scripts/jiahao.py"
      }
    },
    "command": {
      "type": "py-script",
      "entry": "jiahao.py",
      "desc": "装 x 命令"
    }
  }
}
```

- `downloads.scripts` 负责**下载**：将远程脚本保存到 `scripts/jiahao.py`
- `command` 负责**注册**：声明命令类型和入口文件名
- 职责完全分离，多个命令可共享 `downloads.scripts` 中下载的同一文件

:::note

- `downloads.scripts` 字段按声明顺序逐一处理，下载失败立即终止并报错
- 相对路径仅支持 `./xxx` 和 `xxx` 两种写法，`../`、`/xxx`、`~` 不被允许
- 使用 `--cdn` 时，`raw.githubusercontent.com` 的 URL 自动转换为 jsDelivr CDN 加速

:::

#### downloads.bin

下载二进制文件到 `plugins/bin/` 目录。使用嵌套 `map` 对象，**外层 key 为保存的文件名/目录名**，内层通过 `$tar` 标记区分两种模式：

- **直接下载**（`$tar: false`，默认）：下载单文件到 `plugins/bin/<key>`，自动 `chmod +x`
- **压缩包解压**（`$tar: true`）：下载 `.tar.gz`/`.tgz`/`.zip` 到临时文件，`tar -xf` 解压到 `plugins/bin/<key>/` 目录

#### 示例：直接下载二进制（$tar: false）

```json
{
  "my-tool": {
    "downloads": {
      "bin": {
        "my-tool": {
          "linux-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-amd64",
          "linux-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-arm64",
          "darwin-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-amd64",
          "darwin-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-arm64",
          "windows-x64": "https://github.com/xxx/releases/download/v1.0/my-tool-windows-amd64.exe"
        }
      }
    },
    "command": {
      "type": "bin",
      "entry": "my-tool",
      "desc": "My platform-specific binary tool"
    }
  }
}
```

#### 示例：压缩包解压（$tar: true）

```json
{
  "my-tool": {
    "downloads": {
      "bin": {
        "my-tool": {
          "$tar": true,
          "linux-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-x86_64.tar.gz",
          "linux-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-arm64.tar.gz",
          "darwin-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-x86_64.tar.gz",
          "darwin-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-arm64.tar.gz",
          "windows-x64": "https://github.com/xxx/releases/download/v1.0/my-tool-windows-x64.zip"
        }
      }
    },
    "command": {
      "type": "bin",
      "entry": "my-tool/my-tool",
      "desc": "My platform-specific binary tool from archive"
    }
  }
}
```

解压后目录结构：`plugins/bin/my-tool/my-tool`，`entry` 对应 `my-tool/my-tool`。

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `downloads.bin` | `Record<string, PlatformMap>` | 否 | 二进制下载映射：`{ "<文件名/目录名>": { "$tar": bool, "<平台>": "<URL>" } }` |
| `$tar` | boolean | 否 | 标记为压缩包，`true` 时解压而非直接下载。默认 `false` |

#### 支持的平台

| 平台 key | 对应系统 |
|----------|---------|
| `linux-x86_64` | Linux x86_64 |
| `linux-arm64` | Linux ARM64 |
| `darwin-x86_64` | macOS Intel |
| `darwin-arm64` | macOS Apple Silicon |
| `windows-x64` | Windows x64 |

安装时 byk 根据当前平台自动匹配对应的 URL，仅下载匹配的文件。如果当前平台没有对应的 key，直接跳过不安装。

:::note

- `downloads.bin` 的 value 都可以是远程 URL 或相对路径 `./xxx`，解析规则同 Ref 引用
- 使用 `--cdn` 时，`raw.githubusercontent.com` 的 URL 自动转换为 jsDelivr CDN 加速
- `$tar: true` 使用系统 `tar` 命令解压，支持 `.tar.gz` / `.tgz` / `.zip` 格式，要求 Windows 10 1803+（内置 `tar`）
- `$tar: false`（直接下载）在非 Windows 系统自动设置执行权限（`chmod 755`）
- `$tar: true` 不修改压缩包内文件权限，解压后自动记录所有文件/目录，卸载时完整清理
- 解压时一个 key 对应一个独立子目录，**不同插件互不干扰**，卸载时直接删除整个目录
- `bin` 命令入口 `entry` 可以带相对路径（如 `my-tool/bin/my-tool`），不允许 `../` 或绝对路径

:::

### download-scripts（语法糖）

`download-scripts` 是 `downloads.scripts` 的扁平写法，避免多层嵌套。当插件只需要下载脚本时，可以省略 `downloads` 这一层：

```json
{
  "jiahao": {
    "download-scripts": {
      "jiahao.py": "https://raw.githubusercontent.com/fcbyk/pyodd/refs/heads/main/scripts/jiahao.py"
    },
    "command": {
      "type": "py-script",
      "entry": "jiahao.py",
      "desc": "装 x 命令"
    }
  }
}
```

等价于：

```json
{
  "jiahao": {
    "downloads": {
      "scripts": {
        "jiahao.py": "https://raw.githubusercontent.com/fcbyk/pyodd/refs/heads/main/scripts/jiahao.py"
      }
    },
    "command": {
      "type": "py-script",
      "entry": "jiahao.py",
      "desc": "装 x 命令"
    }
  }
}
```

### download-bin（语法糖）

`download-bin` 是 `downloads.bin` 的扁平写法，与 `download-scripts` 同理：

```json
{
  "my-tool": {
    "download-bin": {
      "my-tool": {
        "darwin-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-arm64",
        "linux-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-amd64"
      }
    },
    "command": {
      "type": "bin",
      "entry": "my-tool",
      "desc": "My platform-specific binary tool"
    }
  }
}
```

等价于：

```json
{
  "my-tool": {
    "downloads": {
      "bin": {
        "my-tool": {
          "darwin-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-arm64",
          "linux-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-amd64"
        }
      }
    },
    "command": {
      "type": "bin",
      "entry": "my-tool",
      "desc": "My platform-specific binary tool"
    }
  }
}
```

#### 混用与冲突

`download-scripts` / `download-bin` 可与 `downloads` 混用，但**不能同时定义同名字段**：

| 写法 | 结果 |
|------|------|
| `download-scripts` + `downloads.bin` | ✅ 无冲突，合并执行 |
| `download-bin` + `downloads.scripts` | ✅ 无冲突，合并执行 |
| `download-scripts` + `downloads.scripts` | ❌ 冲突，安装时报错 |
| `download-bin` + `downloads.bin` | ❌ 冲突，安装时报错 |

### command

为单命令插件注册命令，**命令名自动取插件 key**。推荐只有一个命令的插件使用此字段。

```json
{
  "hello": {
    "pip": ["byk-hello"],
    "command": {
      "type": "py-module",
      "entry": "hello.one",
      "desc": "Say hello"
    }
  }
}
```

安装后，用户通过 `byk hello` 即可执行该命令。

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 命令类型：`py-module`（模块）、`py-script`（脚本）、`py-bin`（Python 可执行文件）或 `bin`（普通二进制） |
| `entry` | string | 是 | 入口点：`py-module` 为模块路径，`py-script` 为脚本文件名（需通过 `download-scripts` 或 `downloads.scripts` 预先下载），`py-bin` 为 venv/bin 下的可执行文件名，`bin` 为二进制文件名（需通过 `download-bin` 或 `downloads.bin` 预先下载） |
| `desc` | string | 否 | 命令描述，显示在帮助信息中 |

### commands

声明插件提供的多个命令。`commands` 中的内容会被**零解析**直接合并到 `plugins.cmd.json`，无需中间转换。

```json
{
  "commands": {
    "hello": {
      "type": "py-module",
      "entry": "hello.one",
      "desc": "Example subcommand to verify dynamic registration."
    },
    "world": {
      "type": "py-module",
      "entry": "hello.two",
      "desc": "Second command to verify dynamic registration."
    }
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 命令类型：`py-module`（模块）、`py-script`（脚本）、`py-bin`（Python 可执行文件）或 `bin`（普通二进制） |
| `entry` | string | 是 | 入口点：`py-module` 为模块路径，`py-script` 为脚本文件名（需通过 `download-scripts` 或 `downloads.scripts` 预先下载），`py-bin` 为 venv/bin 下的可执行文件名，`bin` 为二进制文件名（需通过 `download-bin` 或 `downloads.bin` 预先下载） |
| `desc` | string | 否 | 命令描述，显示在帮助信息中 |

#### 命令类型

| 类型 | 执行方式 | 示例 |
|------|----------|------|
| `py-module` | `python -m <entry>` | `"entry": "hello.one"` |
| `py-script` | `python <scripts_dir>/<entry>` | `"entry": "jiahao.py"` |
| `py-bin` | 直接执行 `venv/bin/<entry>` | `"entry": "my-tool"` |
| `bin` | 直接执行 `plugins/bin/<entry>` | `"entry": "my-tool"` 或 `"entry": "my-tool/bin/my-tool"` |

一个插件可以注册多个命令，每个命令对应一个独立的入口点。

> **与 `command` 共存**：`command` 和 `commands` 可以同时使用。`command` 注册的命令名 = 插件 key，`commands` 注册额外的命名命令。但 `commands` 中不能定义与插件 key 同名的子命令，否则安装时报错。

#### py-script 入口点

对于 `py-script` 类型的命令，`entry` 是一个纯文件名（不含路径），指代 `plugins/scripts/` 目录下的脚本文件。脚本文件需通过 `download-scripts`（或 `downloads.scripts`）字段预先下载/拷贝：

```json
{
  "jiahao": {
    "downloads": {
      "scripts": {
        "jiahao.py": "https://raw.githubusercontent.com/fcbyk/pyodd/refs/heads/main/scripts/jiahao.py"
      }
    },
    "command": {
      "type": "py-script",
      "entry": "jiahao.py",
      "desc": "装 x 命令"
    }
  }
}
```

| 字段 | 职责 |
|------|------|
| `downloads.scripts` / `download-scripts` | 下载/拷贝脚本到 `plugins/scripts/<filename>` |
| `commands`/`command` | 注册命令，`entry` 为 `downloads.scripts` 或 `download-scripts` 中定义的文件名 |


:::note

- `plugins.cmd.json` 中始终存储的是纯文件名，脚本下载由 `downloads.scripts` 字段独立处理
- 多个命令可以共享 `downloads.scripts` 中下载的同一脚本文件
- 命令注册不再包含任何下载逻辑，职责单一

:::

#### bin 入口点

对于 `bin` 类型的命令，`entry` 指代 `plugins/bin/` 目录下的二进制文件路径。可以是一个纯文件名（如 `"my-tool"`），也可以带相对路径（如 `"my-tool/bin/my-tool"`）。二进制文件通过 `download-bin`（或 `downloads.bin`）字段预先下载/解压：

```json
{
  "my-tool": {
    "downloads": {
      "bin": {
        "my-tool": {
          "linux-x86_64": "https://github.com/xxx/releases/download/v1.0/my-tool-linux-amd64",
          "darwin-arm64": "https://github.com/xxx/releases/download/v1.0/my-tool-darwin-arm64"
        }
      }
    },
    "command": {
      "type": "bin",
      "entry": "my-tool",
      "desc": "My platform-specific binary tool"
    }
  }
}
```

| 字段 | 职责 |
|------|------|
| `downloads.bin` / `download-bin` | 根据当前平台下载/解压二进制到 `plugins/bin/` |
| `commands`/`command` | 注册命令，`entry` 为 `plugins/bin/` 下的相对路径（不允许 `../` 或绝对路径） |



## Ref 引用

当一个 key 的值为字符串时，byk 会将其视为引用，从该位置拉取另一个 `byk.json`，并在其中查找同名 key。

### URL 引用

如果字符串以 `http://` 或 `https://` 开头，byk 直接将其作为 URL 拉取：

```json
{
  "lansend": "https://raw.githubusercontent.com/fcbyk/byk-lansend/main/byk.json",
  "hello": {
    "pip": ["byk-hello"],
    "commands": { ... }
  }
}
```

当用户执行 `byk add user/repo/lansend` 时，byk 会：

1. 发现 `lansend` 的值是一个 URL
2. 从该 URL 拉取完整的 `byk.json`
3. 在新拉取的内容中查找 key 为 `lansend` 的条目
4. 使用该条目的操作定义执行安装

### 相对路径引用

如果字符串不以 `http` 开头，byk 将其视为**相对路径**，相对于当前 `byk.json` 所在目录解析：

- **远程模式**（`byk add user/repo`）：拼接到 `https://raw.githubusercontent.com/{user}/{repo}/{branch}/` 后
- **本地模式**（`--file`）：相对于当前 `byk.json` 所在目录解析

```json
{
  "hello": "./plugins/python/byk.json"
}
```

当用户执行 `byk add user/repo/hello` 时，byk 会：

1. 发现 `hello` 的值是相对路径 `./plugins/python/byk.json`
2. 拼接为 `https://raw.githubusercontent.com/user/repo/main/plugins/python/byk.json`
3. 拉取该文件，查找 key 为 `hello` 的条目
4. 使用该条目的操作定义执行安装

**路径规则**：

| 写法 | 说明 |
|------|------|
| `./xxx` 或 `xxx` | 相对于 byk.json 所在目录 ✅ |
| `https://...` | 远程 URL ✅ |
| `../`、`/xxx`、`~` | 不允许 ❌ |

:::note

- `./` 前缀可选，`plugins/python/byk.json` 与 `./plugins/python/byk.json` 等效
- URL 引用**不受安装模式影响**，始终直接拉取
- `../` 等路径逃逸写法不被允许，如需引用外部资源请使用完整 URL
- 相对路径引用适用于将插件定义拆分到子目录的场景
- 每个 byk.json 自治：嵌套 byk.json 中的相对路径始终相对于自身所在目录

:::

> 配合 `$var` 使用：`$var` 可以进一步减少 Ref 路径中的重复前缀，详见 [$var — 变量替换](#var--变量替换)。

Ref 适合将插件定义分散到各自仓库或子目录中维护，主仓库只做索引。

## 远程仓库

任何 GitHub 仓库只要在根目录放一个 `byk.json`，即可作为 byk 插件源使用：

```json title="github.com/myuser/my-byk-plugins/blob/main/byk.json"
{
  "my-tool": {
    "pip": ["my-byk-tool"],
    "command": {
      "type": "py-module",
      "entry": "mybyktool.cli",
      "desc": "My custom tool"
    }
  }
}
```

用户安装方式：

```bash
byk add myuser/my-byk-plugins/my-tool    # 指定 key
byk add myuser/my-byk-plugins            # 安装 byk.json 中的默认 key
```

  "hello": {
    "pip": [
      "byk_hello @ https://github.com/fcbyk/byk/releases/download/plugins/byk_hello-2026.6.21-py3-none-any.whl"
    ],

      "hello": {

      }
    }
  },

## 注意事项

- `byk.json` 中的 key 不能包含 `/`，因为 `/` 在 spec 语法中用于分隔仓库地址和 key
- Ref 引用支持 URL（`http(s)://`）和相对路径两种写法，相对路径遵循 `./xxx` 规则
- 一个插件的 `command` 和 `commands` 可以共存，但 `commands` 中不能定义与插件 key 同名的子命令
- 单命令插件推荐使用 `command`，多命令插件使用 `commands`
- 操作块按固定顺序执行：`pip` / `pip-keep` → `downloads` → `commands` / `command`，与声明顺序无关
- 使用 `--cdn` 时，所有 `raw.githubusercontent.com` 的 URL 自动转换为 jsDelivr CDN 加速
- 顶层 `$var`/`$default` 为系统字段，不会被视为有效 plugin key

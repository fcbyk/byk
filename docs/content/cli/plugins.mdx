# 插件系统

byk 通过隔离的 Python 虚拟环境运行插件，不污染系统全局的 pip 包。插件基于 Python 模块或脚本执行，安装后即可作为一级命令使用。

## 工作原理

插件系统依赖 `~/.byk/venv/` 下的 Python 虚拟环境。当你运行 `byk add <user/repo[/key]>` 时：

1. 从远程仓库（或本地文件）拉取插件协议 `byk.json`
2. 查找匹配的插件 key，按固定顺序执行操作块：`download` → `install` → `commands`
3. 在隔离的 venv 中执行 `pip install`（`install` 块）、下载脚本文件（`download` 块）
4. 将命令路由直接合并到 `~/.byk/plugins/plugins.cmd.json`，包追踪信息持久化到 `~/.byk/plugins/plugins.pkg.json`

之后运行 `byk <command>` 时，byk 根据 `type` 字段区分执行方式：
- `py-module`：执行 `python -m <entry>`
- `py-script`：执行 `python <scripts_dir>/<entry>`

## 初始化

首次使用插件功能时，venv 不存在则 byk 会自动创建，也可以手动初始化：

```bash
byk add <any-plugin>   # 首次 add 时自动创建 venv
```

## 安装插件

### 从远程仓库安装

任何 GitHub 仓库只需在根目录放一个 `byk.json` 即可作为 byk 插件源：

```bash
byk add user/repo           # 安装该仓库 byk.json 中的第一个 key
byk add user/repo/my-key    # 安装指定 key
```

### 指定分支

```bash
byk add --branch dev hello   # 从 dev 分支的 byk.json 拉取
```

### 本地安装

```bash
byk add --file ./local.json my-key   # 使用本地 byk.json 文件
```

### 可编辑安装

适合插件开发场景，将本地源码以可编辑模式安装到 venv：

```bash
byk add -e .              # 从当前目录可编辑安装（读取 ./byk.json，安装第一个 key）
byk add -e . hello        # 安装指定 key
byk add -e /path/to/src   # 从指定目录可编辑安装
```

## 运行插件

安装完成后，插件命令自动注册为 byk 的一级命令，直接使用即可：

```bash
byk <command> [args...]
```

所有命令行参数会原样透传给底层的 Python 模块。

```bash title="示例"
byk lansend --target 192.168.1.10 "hello"
# 等价于 venv/bin/python -m byklansend.main --target 192.168.1.10 "hello"
```

## 查看已安装插件

```bash
byk show plugins        # 列出所有已安装插件及其命令和脚本
byk show <command>      # 查询某个命令是否来自插件（含入口点、描述等）
```

## 卸载插件

```bash
byk remove <plugin-key>   # 卸载指定插件
```

卸载流程：删除下载的脚本文件，清除 `plugins.cmd.json` 和 `plugins.pkg.json` 中该插件注册的所有命令。

:::note
卸载插件时**不会卸载 pip 包**，因为一个 pip 包可能被多个插件共享。留在 venv 中不影响其他插件。
:::

## 目录结构

插件相关的数据存储在以下位置：

```tree
~/.byk/
├── venv/               # Python 虚拟环境（python3 -m venv 创建）
│   ├── bin/            # Python 解释器和 pip
│   └── lib/
└── plugins/
    ├── plugins.cmd.json   # 命令路由（热路径：命令解析、帮助渲染）
    ├── plugins.pkg.json   # 包追踪（冷路径：卸载、信息查询）
    └── scripts/           # py-script 脚本文件存放目录
```

## 状态文件

插件系统使用两个 JSON 文件分别管理命令路由和包追踪，由 `byk add` / `byk remove` 自动维护。

### plugins.cmd.json（命令路由）

热路径文件，用于命令解析、路由和帮助渲染：

```json
{
  "commands": {
    "lansend": {
      "type": "py-module",
      "entry": "byklansend.main",
      "desc": "Send messages via LAN"
    },
    "myscript": {
      "type": "py-script",
      "entry": "myscript.py",
      "desc": "My custom script"
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `commands` | 命令名 → 类型、入口点、描述的映射 |
| `type` | 命令类型：`py-module`（模块）或 `py-script`（脚本） |
| `entry` | 入口点：`py-module` 为模块路径，`py-script` 为脚本文件名 |
| `desc` | 命令描述，显示在帮助信息中 |

### plugins.pkg.json（包追踪）

冷路径文件，记录每个插件的安装来源和注册命令，用于卸载和信息查询：

```json
{
  "packages": {
    "hello": {
      "source": "user/repo",
      "install": {
        "pip": ["byk-hello"],
        "pip-e": ["./plugins/python"]
      },
      "commands": ["hello", "world"]
    },
    "jiahao": {
      "source": null,
      "install": {
        "pip": ["rich", "click"]
      },
      "download": {
        "scripts": ["jiahao.py"]
      },
      "commands": ["jiahao"]
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `packages` | 插件 key → 包信息的映射 |
| `source` | 插件来源（`user/repo` 格式），本地安装时为 `null` |
| `install` | 安装依赖：`pip` 为包名/URL 列表，`pip-e` 为可编辑安装路径列表 |
| `download` | 下载资源：`scripts` 为脚本文件名列表 |
| `commands` | 该插件注册的所有命令名列表 |

# 插件协议 byk.json

每个 byk 插件的发布入口是仓库根目录下的 `byk.json` 文件。它定义了插件的安装方式、下载资源和注册的命令。本文档面向插件发布者，说明如何编写这个文件。

## 文件位置

将 `byk.json` 放在 GitHub 仓库的根目录。byk 通过 `raw.githubusercontent.com` 拉取该文件：

```
https://raw.githubusercontent.com/{owner}/{repo}/{branch}/byk.json
```

## 设计哲学

`byk.json` 是一个**过程式操作脚本**，按顺序声明三个操作块：

```
1. download → 下载远程资源到本地
2. install  → 安装依赖到虚拟环境
3. commands → 注册命令到路由表
```

JSON key 的声明顺序即执行顺序，阅读即执行。

## 整体结构

`byk.json` 是一个 JSON 对象，每个顶级 key 为一个**插件 key**（即 `byk add <key>` 中的 `<key>`），值为该插件的操作定义：

```json
{
  "<plugin-key>": {
    "install": { ... },
    "download": { ... },
    "commands": { ... }
  }
}
```

一个 `byk.json` 可以包含多个插件 key，用户通过 `byk add <key>` 指定要安装哪个。

## 操作块

### install — 安装依赖

安装依赖到 byk 的隔离虚拟环境。当前支持 `pip`（PyPI 包名 / URL）和 `pip-e`（可编辑安装）。

```json
{
  "hello": {
    "install": {
      "pip": ["byk-hello"],
      "pip-e": ["./plugins/python"]
    },
    "commands": { ... }
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pip` | string[] | 否 | pip 包名或 URL 列表，执行 `pip install <item>` |
| `pip-e` | string[] | 否 | 可编辑安装路径列表，执行 `pip install -e <dir>/<path>`（仅在 `-e` 模式下生效） |

`pip` 数组中的每一项可以是：
- **包名**：`"rich"`、`"click>=8.0"`、`"requests==2.28.0"` — 从 PyPI 安装
- **URL**：`"https://github.com/.../xxx.whl"` — 从远程 URL 安装

### download — 下载资源

从远程 URL 下载文件到本地 `~/.byk/plugins/scripts/` 目录。文件名由 URL 最后一个 `/` 之后的部分决定。

```json
{
  "myscript": {
    "download": {
      "scripts": [
        "https://raw.githubusercontent.com/fcbyk/pyodd/refs/heads/main/scripts/jiahao.py"
      ]
    },
    "commands": { ... }
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `scripts` | string[] | 否 | 脚本文件 URL 列表，下载到 `~/.byk/plugins/scripts/` |

:::note
下载目录不可自定义，所有脚本统一放在 `scripts/` 目录下，避免文件污染。文件名由 URL 自动提取。
:::

### commands — 注册命令

声明插件提供的命令。`commands` 中的内容会被**零解析**直接合并到 `plugins.cmd.json`，无需中间转换。

```json
{
  "commands": {
    "hello": {
      "type": "py-module",
      "entry": "hello.one",
      "desc": "Example subcommand to verify dynamic registration."
    },
    "world": {
      "type": "py-module",
      "entry": "hello.two",
      "desc": "Second command to verify dynamic registration."
    }
  }
}
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 命令类型：`py-module`（模块）或 `py-script`（脚本） |
| `entry` | string | 是 | 入口点：`py-module` 为模块路径，`py-script` 为脚本文件名 |
| `desc` | string | 否 | 命令描述，显示在帮助信息中 |

#### 命令类型

| 类型 | 执行方式 | 示例 |
|------|----------|------|
| `py-module` | `python -m <entry>` | `"entry": "hello.one"` |
| `py-script` | `python <scripts_dir>/<entry>` | `"entry": "jiahao.py"` |

一个插件可以注册多个命令，每个命令对应一个独立的入口点。

## Ref 引用

如果一个 key 的值为字符串而非对象，byk 会将其视为 **URL 引用**——从该 URL 拉取另一个 `byk.json`，并在其中查找同名 key：

```json
{
  "lansend": "https://raw.githubusercontent.com/fcbyk/byk-lansend/main/byk.json",
  "hello": {
    "install": {
      "pip": ["byk-hello"]
    },
    "commands": { ... }
  }
}
```

当用户执行 `byk add user/repo/lansend` 时，byk 会：

1. 发现 `lansend` 的值是一个 URL
2. 从该 URL 拉取完整的 `byk.json`
3. 在新拉取的内容中查找 key 为 `lansend` 的条目
4. 使用该条目的操作定义执行安装

Ref 适合将插件定义分散到各自仓库中维护，主仓库只做索引。

## 远程仓库

任何 GitHub 仓库只要在根目录放一个 `byk.json`，即可作为 byk 插件源使用：

```json title="github.com/myuser/my-byk-plugins/blob/main/byk.json"
{
  "my-tool": {
    "install": {
      "pip": ["my-byk-tool"]
    },
    "commands": {
      "my-tool": {
        "type": "py-module",
        "entry": "mybyktool.cli",
        "desc": "My custom tool"
      }
    }
  }
}
```

用户安装方式：

```bash
byk add myuser/my-byk-plugins/my-tool    # 指定 key
byk add myuser/my-byk-plugins            # 安装 byk.json 中第一个 key
```

## 完整示例

一个包含多个插件、使用 Ref 引用和完整操作块的 `byk.json`：

```json
{
  "lansend": "https://raw.githubusercontent.com/fcbyk/byk-lansend/main/byk.json",

  "hello": {
    "install": {
      "pip": [
        "https://github.com/fcbyk/byk/releases/download/plugins/byk_hello-2026.6.21-py3-none-any.whl"
      ],
      "pip-e": ["./plugins/python"]
    },
    "commands": {
      "hello": {
        "type": "py-module",
        "entry": "hello.one",
        "desc": "Example subcommand to verify dynamic registration."
      },
      "world": {
        "type": "py-module",
        "entry": "hello.two",
        "desc": "Second command to verify dynamic registration."
      }
    }
  },

  "jiahao": {
    "install": {
      "pip": ["rich", "click"]
    },
    "download": {
      "scripts": [
        "https://raw.githubusercontent.com/fcbyk/pyodd/refs/heads/main/scripts/jiahao.py"
      ]
    },
    "commands": {
      "jiahao": {
        "type": "py-script",
        "entry": "jiahao.py",
        "desc": "装 x 命令"
      }
    }
  }
}
```

## 注意事项

- `byk.json` 中的 key 不能包含 `/`，因为 `/` 在 spec 语法中用于分隔仓库地址和 key
- Ref 引用的 URL 必须以 `http://` 或 `https://` 开头
- 一个插件的 `commands` 中可以注册多个命令，但命令名不要与其他插件冲突，否则会按安装顺序覆盖
- 远程仓库的 `byk.json` 中如果有多个 key，用户可通过 `user/repo/key` 精确指定要安装的插件
- 操作块按固定顺序执行：`download` → `install` → `commands`，与声明顺序无关
